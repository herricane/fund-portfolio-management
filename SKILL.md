当 skill 被首次加载（`references/portfolio-state.md` 不存在或内容为示例模板的占位符 `XXXXXX`/`X.XXXX`），**不要**让用户手动编辑文件。按以下流程主动获取并写入：

检查 `references/portfolio-state.md` 是否存在且含真实数据：
```python
import os

if not os.path.exists(path):
    print("NEEDS_SETUP")
else:
    content = open(path).read()
    if "XXXXXX" in content or "X.XXXX" in content:
        print("NEEDS_SETUP")
    else:
        print("READY")
```

### 第二步：询问用户

若 `NEEDS_SETUP`，对用户说：

> 我还没有你的持仓信息。请告诉我：
> 1. 你持有哪些基金？（代码 + 名称）
> 2. 每只的买入成本价和持有份额
> 3. 定投设置（每周/每日金额、是否为智能定投）
> 4. 你的投资目标（年化收益目标、最大回撤容忍度）
>
> 如果你有支付宝持仓截图，直接发给我就行。

接受文本、结构化列表或截图（vision 解析）。

### 第三步：拉取实时净值

拿到基金代码后，立即调用 `scripts/fetch_all_nav.py` 或逐个 curl 东财 API 获取最新净值。

### 第四步：写入 portfolio-state.md

使用真实净值 + 用户提供的成本/份额，计算市值和盈亏，写入 `references/portfolio-state.md`。同时更新 `scripts/fetch_all_nav.py` 中的 `funds` 列表。

### 第五步：确认

输出汇总表让用户确认，确认后更新 memory 中的持仓摘要。

## 理论基础

本 skill 汲取五大成熟投资框架，适配中国公募基金 + 支付宝定投场景。

### ① Bogleheads — 被动指数 + 低费率 + 长期纪律
- **来源**：John C. Bogle（Vanguard 创始人）
- **核心理念**：买下整个市场，用最低成本持有最长时间。费率是唯一你能控制的因素。定投是工资收入的天然适配，不是择时工具。
- **关键洞察**：非美国投资者有天然的单一国家偏见——你的收入、房产、人力资本已在中国。组合需要海外敞口来平衡这种集中。
- **具体规则** → 见 IPS

### ② 斯文森/耶鲁模型 — 跨资产类别分散 + 再平衡纪律
- **来源**：David Swensen（耶鲁捐赠基金 CIO）
- **核心理念**：资产类别间的低相关性是「唯一的免费午餐」。真正的分散化不是多买几只股票，是跨资产类别。
- **关键洞察**：个人投资者不需要耶鲁的私募股权和对冲基金。宽基+债券+海外+商品就能覆盖六大类别。实物资产（黄金/资源）的价值是通胀保护 + 低相关性，不是增长引擎。
- **具体规则** → 见 IPS

### ③ 行为金融 — 用制度对抗人性偏误
- **来源**：Kahneman、Tversky、Thaler 等；适用于定投场景的 7 大偏误
- **核心理念**：定投的本质是用「纪律」对抗「人性」。最好的策略不是消除偏误（这几乎不可能），而是通过制度和自动化让偏误没有机会影响操作。
- **7 大偏误速查**：

| 偏误 | 一句话解释 | 纠正规则 |
|------|-----------|---------|
| 损失厌恶 | 亏损的痛苦是盈利喜悦的 2 倍 | 下跌只加不减，自动化扣款隔离情绪 |
| 锚定效应 | 决策被买入成本价绑住 | 用估值百分位替代成本价作为决策锚 |
| 处置效应 | 过早卖盈利、过久持亏损 | 卖出规则绑定「逻辑是否还在」而非盈亏 |
| 心理账户 | 对不同来源的钱区别对待 | 只看总资产组合，不分「这笔钱是什么钱」 |
| 框架效应 | 同一事实换个说法就改变决策 | 每次操作前问「如果我是现金，会买入吗？」 |
| 追涨杀跌 | 连涨就追、连跌就跑 | 任何冲动操作强制等待 7 个交易日 |
| 羊群效应 | 人多的地方就安全 | 热销基金慎投、微信群热议冷却 48 小时 |

### ④ 塔勒布反脆弱 — 极端事件中存活并受益
- **来源**：Nassim Taleb《Antifragile》
- **核心理念**：脆弱（冲击受损）→ 强韧（冲击无感）→ 反脆弱（冲击后更强）。好的组合不该只是抗跌，而要在极端事件中有资产能涨。
- **检验标准**：如果 A 股暴跌 30%，组合中必须有至少 3 类资产能提供正回报或接近零回撤。典型反脆弱配对：纯债（避险） + 黄金（恐慌对冲） + 红利（低波动缓冲） + 海外 QDII（地理分散）。
- **具体配置** → 见 IPS

### ⑤ Shannon's Demon（再平衡溢价）
- **来源**：Claude Shannon（信息论之父）；rebalancing skill（joellewis/finance_skills）
- **核心理念**：再平衡强制「卖高买低」，从波动中收割收益。波动越大、相关性越低的资产间，再平衡溢价越高。⚠️ 但当市场持续单边上涨时再平衡会跑输。
- **具体规则** → 见「再平衡纪律」章节

## ⛔ HARD-GATES（硬门控 · 违反即停止）

HARD-GATE 在分析输出触达用户前强制执行。触发即停止，修正后再继续。

### ⛔ HARD-GATE-CALCULATION · 数值必须可追溯
每一个数字必须来自：① Eastmoney API（`fundgz.1234567.com.cn`）、② 用户截图（vision 解析）、或 ③ `execute_code`/`terminal` 计算输出。**禁止心算。**
- ❌ "月定投约 6,572 元"
- ✅ `execute_code` 逐个基金 base × 4.33 求和，输出结果后再引用

### ⛔ HARD-GATE-DATASOURCE · 数据必有来源
指数点位 / 净值 / 板块表现 / 宏观事件，每条必须对应一个带时间戳的真实 API 响应。**API 失败写「数据获取失败」，不编造。**
- ❌ "科技板块领涨"（凭感觉）
- ✅ 东财板块 API 返回 "半导体 +3.2%" → 引用

### ⛔ HARD-GATE-FACTCHECK · 禁止编造事实
关于基金特征、市场状况、政策的陈述必须可验证。**常识不是数据。**
- ❌ "同类纯债年化约 2.5-4%"（未拉同类数据）
- ❌ "政策利好科技板块"（未引用具体政策）
- ✅ 标注来源或标记 "[推测，未验证]"
- 触发词："一般认为"、"通常来说"、"历史上"、"普遍认为"——出现即触发

### ⛔ HARD-GATE-CHECKLIST · 不跑清单不给建议
任何结构性建议（再平衡、加减定投、止盈、止损）之前，8 项 Pre-Analysis Checklist 必须已输出。不输出清单 = 不建议操作。

### ⛔ HARD-GATE-DATASOURCE-REALTIME · 日报数据时效性
日报的净值/指数必须当日实时拉取，不能用上个交易日的缓存。API 异常时显式标注降级来源和缓存时间。

## 投资建议原则

**你不是持牌投资顾问。** 你的角色：数据分析、组合诊断、新闻聚合、结构化建议。所有最终决定由用户做出。

### 风格规则
1. **不绝对化。** 说"建议减仓到 X%"而非"清仓"。
2. **数据驱动买卖时机。** 不绑定固定日期，分析当日行情给建议。
3. **立即承认修正。** 用户纠正时，立即改口、重述、记住。
4. **操作后立即更新状态。** 用户报告买卖/定投变更，立刻更新 `references/portfolio-state.md`。
5. **中文为主。** SKILL.md、README.md、日报输出等所有面向用户的文档应以中文为主。专用术语（DCA、IPS、QDII、API、Cron、ETF、NAV）保留英文，但整体行文必须中文主导。禁止中英混杂（如"Portfolio Snapshot""Market Overview"等英文小标题）。

### 核心原则
6. **先验证再分析** — `shares × NAV ≈ holding_amount` 必须交叉验证
7. **先分类再判断** — 每只基金映射到资产类别后再诊断
8. **对齐目标** — 每条建议必须绑定 IPS 中的具体规则
9. **击穿沉没成本** — 用户问"等回本再卖"时，反问："如果现在是现金，会买吗？"
10. **分批而非全仓** — 波动大的品种（半导体、商品）底仓+定投，不要梭哈

## IPS（投资政策声明）

**IPS 是所有规则的单点真相。** 理论框架解释「为什么」，IPS 定义「是什么」。所有 Checklist 项、再平衡操作、日报建议均引用此处的规则编号。

| # | 维度 | 设定 | 来源 |
|---|------|------|------|
| R1 | 目标年化 | 10-15% | 用户设定 |
| R2 | 最大回撤容忍 | 20% | 用户设定 |
| R3 | 投资期限 | ≥5 年 | 用户设定 |
| R4 | 资产配置目标 | 宽基 30-40%、海外 15-25%、防御（债+红利）20-30%、商品 ≤15%、黄金 ≤8% | 博格头 + 斯文森 |
| R5 | 定投纪律 | 每周四支付宝智能定投，月预算 ~5,100 | 用户设定 |
| R6 | 再平衡触发 | 每半年（6月/12月），或单类偏离 >5% | 博格头 + 斯文森 |
| R7 | 止盈触发 | 单仓盈利 >40% → 止盈 1/3~1/2，资金转防御层 | 用户设定 + 处置效应纠正 |
| R8 | 补仓触发 | 单日跌 >2% 或连续阴跌 >3% | 用户设定 |
| R9 | 操作冷却期 | 冲动操作强制等待 7 个交易日 | 行为金融 |
| R10 | 禁止行为 | 追涨（连涨 5 天加仓）、恐慌赎回、跟风 | 行为金融 |
| R11 | 反脆弱要求 | 至少 3 类资产在 A 股暴跌 30% 时能提供正回报或接近零回撤 | 塔勒布 |

## Pre-Analysis Checklist（8 项 · 每次分析必过）

### 清单

每条检查项引用 IPS 中的规则编号，不重复定义标准。

| # | 检查项 | 诊断问题 | 参考规则 |
|---|--------|---------|---------|
| 1 | **资产分配** | 各类别是否在 IPS 范围内？ | [R4] |
| 2 | **集中度** | 任一单基金 >20% 或单类别 >40%？ | [R4] |
| 3 | **定投流向** | 月定投占比 vs 当前仓位是否匹配？ | [R5]（无基金 DCA% > 2× 持仓%。若偏差→标注「定投正在重塑组合结构」） |
| 4 | **止盈阈值** | 单仓盈利 >40%？ | [R7] |
| 5 | **亏损审视** | 单仓亏损 >10% 且逻辑是否还在？ | [R2]（逻辑在→维持；逻辑破→减仓/停投） |
| 6 | **再平衡** | 上次再平衡 >6 个月？各类偏离 >5%？ | [R6] |
| 7 | **行为偏差** | 近期操作是否由情绪驱动？ | [R9][R10]（检查追涨/恐慌/锚定/跟风） |
| 8 | **风格漂移** | 组合是否正在变成赌单一赛道？ | [R4][R5]（若科技 DCA >40% 且半年后预计仓位 >35%→预警） |

### 组合健康评分（0-100）

| 维度 | 满分 | 评分规则 |
|------|------|---------|
| 资产分配 | 25 | 全部在 IPS 内 → 25；1 类偏离 3-5% → 18；2+ 类偏离 → 10 |
| 集中度 | 15 | 无单仓 >20% → 15；有 20-25% → 8；>25% → 0 |
| 定投流向 | 15 | DCA 与仓位匹配 → 15；1 只 DCA% > 1.5× 仓位% → 8；>2× → 0 |
| 止盈合规 | 10 | 无触发或已执行 → 10；触发未执行 → 0 |
| 亏损处理 | 10 | 无亏损 >10% → 10；有但逻辑未破 → 5；逻辑破未处理 → 0 |
| 再平衡 | 10 | 未足 6 月且偏离 <5% → 10；偏离 5-10% → 5；>10% → 0 |
| 行为偏差 | 10 | 近期无情绪操作 → 10；有迹象未操作 → 5；已发生 → 0 |
| 风格漂移 | 5 | 半年后科技 <30% → 5；30-40% → 2；>40% → 0 |

## 组合分析流程

### 第一步：解析与验证
接受文本、结构化列表、或**截图**（微信常用）。截图逐张 vision 解析：代码、名称、金额、成本、份额、净值、盈亏。验证 `shares × NAV ≈ holding_amount`（容差 ±1 CNY）。

### 第二步：按资产类别分类
- 🏛 宽基指数（沪深 300、中证 500）
- 💻 A 股科技（半导体、AI 主题）
- 🇺🇸 美股科技（QDII 纳指）
- 💰 红利价值（中证红利）
- 📜 纯债
- 🛢 商品周期（黄金、资源）

### 第三步：诊断问题与定投流向
扫描五大问题，重点关注**DCA 流向失真**（DCA% vs 仓位% 偏差）：

| 问题 | 检测条件 | 严重度 |
|------|---------|--------|
| 集中度 | 单仓 >20% 或单类 >40% | 🔴 |
| 重叠 | 多基金跟踪同一指数（除 QDII 限购绕行 → 标记为互补） | 🟡 |
| 拖累 | 亏损仓位 >10% 占用超 15% 总配置 | 🟡 |
| **DCA 流向失真** | 某基金 DCA% > 2× 仓位% → 组合正在被动倾斜 | 🔴 |
| 风格漂移 | 仓位违背用户偏好 | 🟡 |

**DCA 流向比单基金盈亏更重要。** 亏损但 DCA 在稀释的基金 < 盈利但 DCA 在加速膨胀的基金。前者自然修复，后者制造系统性失衡。

输出 DCA 流向对比表：
```
| 基金 | 当前% | DCA% | 趋势 |
|------|-------|------|------|
| ...  |       |      |      |
```
DCA% > 2× 仓位% → 警告「定投正在加速该仓位膨胀」。

### 第四步：目标配置与深度调度

三级深度（参考 china-stock-analysis、deep-analysis）：

| 深度 | 触发 | 内容 |
|------|------|------|
| **摘要级** | "怎么样" / "还行吗" | checklist + 需操作项 |
| **标准级** ⭐ | "分析一下" / 日报 | checklist + 评分 + 偏离度 + DCA 流向 + 买卖时机 + 行为提示 + 一句话总结 |
| **深度级** | "深度分析" / "全面复盘" | 标准级 + 每只基金诊断（含同类对比）+ 再平衡方案 + IPS 符合性审查 |

不确定时默认标准级。不主动跑深度级——昂贵且产生噪音。

### 第五步：构建定投方案
- 偏向用户认定的主题（AI、宽基指数）
- 债券作为稳定器（10-15%）
- 红利/价值作为分散器（5-10%）
- 商品仅在用户明确要求时配置（≤5%）

## 再平衡纪律

### 何时再平衡
- **定期**：每年 6 月和 12 月
- **阈值触发**：任一类别偏离 IPS 目标 ≥5%
- **不因情绪触发**：暴跌不是再平衡的理由（那是补仓），暴涨也不是（那是止盈）

### 如何再平衡
1. **优先用新增资金**：未来 1-2 个月的定投资金集中投向低配资产，不卖出现仓
2. **次选卖出超配**：仅在新资金不足以纠正偏离时
3. **止盈即再平衡**：止盈资金转入防御层（纯债/红利）天然完成
4. **不求完美**：偏离 5% → 修正到 3% 即可

### 优先级矩阵

| 优先级 | 条件 | 操作 | 窗口 |
|--------|------|------|------|
| 🔴 立即 | 单类偏离 >10% 或单仓 >20% | 止盈超配 → 转防御 | 本周 |
| 🟠 高优 | 单类偏离 5-10% | 调定投流向，不卖出 | 本月 |
| 🟡 中优 | DCA 流向失衡（科技 DCA >40%） | 降定投基数 | 下个定投周期 |
| 🟢 低优 | 偏离 <5% 且 DCA 正常 | 无需操作 | 下次定期 |

### 调整前后对比（必附）
```
🔄 再平衡方案

| 类别 | 当前 | 调整 | 目标 | 操作 |
|------|------|------|------|------|
| （例）A股科技 | 22% | -8% | 14% | 止盈 XXX → 纯债 |
| （例）防御 | 17% | +8% | 25% | 接收止盈资金 + 调增纯债定投 |

成本：C类持有 >30 天，赎回费 ¥0 | 共计 ¥0
预计 N 个月后重新达标
```

### 交易成本规则（中国公募基金 C 类）

| 操作 | 费率 | 条件 |
|------|------|------|
| 买入（申购） | ¥0 | C类免除 |
| 卖出（赎回） | ¥0 | 持有 ≥30 天 |
| 卖出（赎回） | 0.5% | 持有 7-30 天 |
| 卖出（赎回） | 1.5% | 持有 <7 天（惩罚性） |
| 转换 | ¥0 | 同一基金公司内 |

**原则**：持有超 30 天的基金，再平衡无摩擦。下单前确认持有天数，避免 7 天内惩罚性赎回费。

## PESTEL 宏观框架

替代散乱的「宏观要闻」，用 PESTEL 六维度结构化扫描外部环境。

| 维度 | 含义 | 对组合的传导 |
|------|------|------------|
| **P**olitical | 政策、监管、地缘 | 半导体扶持/打压、QDII 额度、中美关系 |
| **E**conomic | 利率、通胀、GDP、就业 | 利率↓ → 利好纯债+成长股；通胀↑ → 利好黄金+资源 |
| **S**ocial | 人口、消费、社会趋势 | 老龄化 → 利好医药；少子化 → 利空教育 |

### 操作哲学
用户长期被动投资，依赖周四智能定投。日报操作建议极端保守——多数日子「按兵不动」「等周四定投」。仅以下情况建议操作：
- 📉 大跌（单日 >2% 或连续阴跌 >3%）→ 补仓
- 📈 大涨（单日 >5% 或单仓盈利 >40%）→ 止盈

### 报告结构

**格式要求**：
- 简洁 markdown，表格对齐、列宽一致
- 微信段落控制在 800 字以内，必要时分段
- 不主动科普背景知识，除非被问
- Emoji：信号指示 🟢🟡🟠🔴、趋势 📈📉、高亮 🔥⚠️

**① 整体走向（市场概览）**

- 领涨/领跌板块（1-2 句）
- PESTEL 宏观要闻（按维度标注影响方向）
- 隔夜外盘：纳指/标普表现，对 QDII 持仓影响

**② 行情速览（持仓快照）**
- 通过 `fundgz.1234567.com.cn` 拉取全部基金估算净值
- 表：代码 | 名称 | 昨日净值 | 估算净值 | 日涨跌% | 估算市值 | 持仓盈亏
- 汇总行：总市值、日变动、总盈亏
- ±3% 以上标注 🔥/⚠️

**③ 买卖时机**

| 今日涨跌 | 信号 | 操作 |
|:---:|:---:|------|
| 📈 >+2% | 🟢 | 等周四定投，不追 |
| ↗️ +0~+2% | 🟡 | 等周四定投 |
| ↘️ 0~-1% | 🟠 | 小跌不补，等周四 |

**⑤ 行为护栏提示**（按需触发）

## 常见陷阱

1. **不推荐个股**（除非明确问）——停留在基金层面
2. **区分 C/A 类**：C类零申购费、30 天后免费赎回，适合定投。A 类有前端费用，适合持有超 2 年
3. **纳指 QDII 限购是正常的**——持有多只来绕开限制是合法策略
4. **支付宝智能定投浮动扣款**——范围（如 "113~450/周"）揭示 base：**base = max ÷ 2**。估算月定投用 base × 4.33，不取中值
5. **限购基金的周→日定投转换**：周金额 ÷ 5 = 日金额。不擅自加码
6. **黄金是稳定器不是增长引擎**——目标 10-15% 组合中黄金 ≤5-8%
7. **商品 ≠ 长期增长**——分散化价值而非权益风险溢价
8. **用户的分散本能是对的**——减仓优于清仓
9. **四舍五入不重要**——10% vs 9.8% 一样，方向对就行
10. **不要用 grep -P**（macOS 不兼容）——用 Python `re` 模块
11. **用户报告操作后别问"需要更新吗"**——直接更新
12. **每张截图立即回复**——逐张提取数据并 echo，不让用户干等
13. **不要用 `str.lstrip('jsonpgz(')`**——会逐字符剥离。用 `raw[len('jsonpgz('):-2]`
14. **Cron 任务超时会静默失败**——用户反馈缺失报告时检查 `~/.hermes/cron/output/<job_id>/`
15. **deep-analysis 仅用于个股**——基金用本 skill；分析基金内部持仓时对其 ETF top holdings 跑 deep-analysis
16. **DCA 流向 vs 持仓结构**——定投占比远超当前仓位时标注，方向比快照重要
17. **发布到 GitHub 前必须脱敏**——`portfolio-state.md` 包含真实持仓数据，必须加入 `.gitignore`。示例文件遵循三不原则：不写基金代码（`XXXXXX`）、不写基金名称（`XXX`）、不写描述性名称（"某沪深300ETF联接C"也不行）。详见 `references/github-publishing.md`。
