# 中国公募基金组合管理

## 触发条件

用户提到以下任一关键词时自动加载：
- **操作类**：分析持仓、再平衡、定投、补仓、调整仓位、止盈、止损
- **监控类**：基金日报、基金周报、净值、涨跌
- **规划类**：投资目标、资产配置、风险承受
- **数据类**：分享基金截图、买入卖出确认

## 首次使用 · IPS 初始化

当 `references/portfolio-state.md` 不存在或含占位符 `XXXXXX` 时，按以下流程主动获取。

### 第一步：询问用户

向用户提问（逐条，每条确认后继续）：

> **① 投资期限**：计划持有多久？（A. <3 年 / B. 3-5 年 / C. 5-10 年 / D. >10 年）
>
> **② 收益目标**：期望的年化回报率？（建议参考：纯债 3-4%、平衡 6-8%、偏股 8-10%）
>
> **③ 最大回撤**：你能承受的最大亏损幅度？（下跌多少%会让你想清仓？建议参考：保守 10-15%、适中 20-25%、进取 30-35%）
>
> **④ 月定投能力**：每月大约能投入多少？
>
> **⑤ 当前持仓**：有哪些基金？请提供代码 + 成本价 + 份额 + 定投设置（或直接发截图）
>
> **⑥ 流动性需求**：未来 3 年内有没有大额支出的计划？（买房、结婚、留学等）



**基金分类辅助**：拿到用户持仓后，按以下规则自动分类，分类不确定时向用户确认。

| 方法 | 规则 | 示例 |
|------|------|------|
| 名称关键词 | "沪深300/中证500/创业板/上证50/A500/宽基" → 宽基；"纳斯达克/标普/恒生/QDII/全球" → 海外 QDII；"纯债/债券/利率债/信用债/中短债" → 纯债；"红利/股息" → 红利策略；"半导体/芯片/AI/科技" → 行业主题-科技；"医药/医疗/生物/创新药" → 行业主题-医药；"消费/白酒/食品" → 行业主题-消费；"新能源/光伏/电车/碳中和" → 行业主题-新能源；"黄金/金" → 黄金；"资源/有色/原油/能源" → 商品；"货币/现金/余额" → 货币基金；"可转债/转债" → 可转债 |
| 代码前缀 | 16XXXX → QDII；50XXXX/51XXXX/159XXX → ETF；58XXXX → 科创 50 |
| 用户确认 | 名称模糊（如"战略新兴""优选混合"）→ 主动列出分类选项让用户选择 |

### 第二步：矛盾检测

收到用户回答后，**必须**在写入 IPS 前检查以下矛盾：

| 检测项 | 条件 | 矛盾时输出 |
|--------|------|-----------|
| 收益 vs. 回撤 | 目标年化 >10% 且最大回撤 <20% | ⚠️ 历史上年化 10%+ 的组合最大回撤约 30-40%。建议调高回撤容忍到 ≥25%，或降低收益目标到 6-8%。 |
| 收益 vs. 资产结构 | 目标年化 >8% 但防御资产（债+红利）>40% | ⚠️ 防御资产超 40% 的组合年化通常在 5-7%。要么降防御比例，要么降预期。 |
| 时间 vs. 股票仓位 | 投资 <5 年但股票类 >70% | ⚠️ 5 年内需要的钱不应大量投入股票。建议股票类降至 50% 以下。 |
| 定投 vs. 目标 | 月投金额无法在合理时间内达成目标 | ⚠️ 按当前定投速度，达到目标需要 X 年，超出投资期限。建议增加定投金额或调整目标。 |
| 流动性 | 有短期大额支出但组合以股票为主 | ⚠️ 短期需要的资金不应暴露于股市波动。建议单独建立货币基金缓冲池。 |

**矛盾优先处理**：发现矛盾时，让用户选择改哪个参数。不确定时，选更保守的参数。

### 第三步：写入 IPS

将确认后的参数写入 `references/portfolio-state.md`（参考 `portfolio-state.example.md` 模板）。

```python
# 示例输出（写入 portfolio-state.md 的 IPS 段）
## IPS（投资政策声明 · 2026-06-21 确认）

| # | 维度 | 设定 | 来源 |
|---|------|------|------|
| R1 | 投资期限 | ≥10 年 | 用户 |
| R2 | 收益目标 | 8-10% 年化 | 用户 |
| R3 | 最大回撤容忍 | 25% | 用户 |
| R4 | 资产配置目标 | （示例）宽基 30-40%、海外 15-25%、防御 20-30% —— 实际值由用户②③步输入推导 | R2+R3 推导 |
| R5 | 定投纪律 | ~5,000/月 | 用户 |
| R6 | 再平衡触发 | 每半年 + 单类偏离 >5% | 博格头 + 斯文森 |
| R7 | 止盈触发 | 单类超配 >5% 且无法以新资金纠正 | 卫星层规则 |
| R8 | 操作冷却期 | 冲动操作强制等 7 个交易日 | 行为金融 |
| R9 | 禁止追涨杀跌 | 连涨不加仓、大跌不赎回、不跟风 | 行为金融 |
| R10 | 反脆弱约束 | 组合中至少 2 类资产可在系统性危机中提供正回报或缓冲 | 塔勒布 |
```

## 架构：核心-卫星双层模型

```
核心层 80-90%（永不择时·永续生效）
├─ 定投纪律 — 自动化扣款，不暂停
├─ 资产配置 — 类别目标区间，不漂移
└─ 再平衡日历 — 每半年/阈值触发

卫星层 10-20%（有条件触发·受核心层约束）
├─ 止盈 — 配置偏离触发（非盈亏%触发）
├─ 补仓 — 仅在逻辑未破 + 配置未超限时
└─ 新资金倾斜 — 优先低配资产（非"看好"的赛道）

冲突解决: 核心层 > 卫星层
除非用户明确授权覆盖（需在对话中确认）
```

---

## 核心层 — 始终生效

### 定投纪律

- 定投 = 用自动化对抗情绪。下跌 = 更多份额。**永不暂停。**
- 支付宝智能定投的浮动扣款范围揭示 base：**base = max ÷ 2**。其他平台的定投参数按用户提供值直接使用，不做换算。
- 限购基金的频次转换：周金额 ÷ 5 = 日金额（或按用户指定的交易日数），不擅自加码。

### 资产配置目标

配置目标由用户 IPS（R4）定义。任何建议引用 R4，不重复定义数字。

**防御层定义**：防御层 = 纯债 + 货币基金 + 红利策略。这三类资产的共性是低波动 + 正息收/股息。黄金和商品不属于防御层——它们波动大、无现金流，属于另类资产。

### 再平衡

**何时触发**（R6）：
- 日历触发：每半年一次（用户自定日期）
- 阈值触发：任一类别偏离目标 >5%

**如何执行**：
1. 优先用未来 1-2 月新增资金投向低配资产（零成本）
2. 新资金不足以纠正时，卖出超配 → 买入低配
3. 不求完美：偏离 5% → 修正到 3% 即可
4. 持有 ≥30 天的 C 类基金赎回费 ¥0（摩擦为零）

**优先级矩阵**：

| 优先级 | 条件 | 操作 | 窗口 |
|--------|------|------|------|
| 🔴 立即 | 单类偏离 >10% | 止盈超配 → 转防御 | 本周 |
| 🟠 高优 | 单类偏离 5-10% | 调定投流向 | 本月 |
| 🟡 中优 | 定投流向失衡 | 降定投基数 | 下个周期 |
| 🟢 低优 | 偏离 <5% | 无需操作 | 下次定期 |

## 卫星层 — 有条件触发

卫星层所有操作**必须**满足三个约束，否则退回核心层：
1. 触发条件明确（写在 IPS 里，不是"感觉"）
2. 仓位上限锁死（卫星操作涉及仓位 ≤ 总组合 20%）
3. 有完成标准（不是无限期等待）

### 止盈（R7）

**触发条件**：单类别偏离目标 >5%，且新资金无法在 3 个月内纠正。

**不是止盈的理由**："涨太多了""落袋为安""预感要跌"。

**止盈后资金去向**：优先转入低配的防御层（纯债/红利），而非闲置或投向热门赛道。

**输出要求**：
```
🔄 调整方案
| 基金 | 卖出份额 | 金额(估) | 理由 |
|------|---------|---------|------|
| XXX  | XXX     | ~XXX    | XX类别仓位从 XX%→目标 XX%，偏离 >5%触发 R7 |

→ 资金转入：XXX 纯债 / XXX 红利（防御层低配 -8%）
→ 持有天数确认：≥30 天 ✓ | 费用合计：¥0
```

### 补仓

**触发条件**：单仓亏损 >10%，且满足以下全部条件：
- ✅ 配置未超限（补仓后该类仍在目标区间内）
- ✅ 逻辑未破（该资产类别的投资逻辑仍成立）
- ❌ 单日跌幅 —— 不是触发条件

**流程**：先检查逻辑是否还在，再看仓位是否允许，最后才建议是否加仓。两步都应输出"逻辑检查"和"仓位检查"结论。

### 新资金倾斜

每月定投资金优先投向**当前低配**的资产类别，而非"看好"的赛道。

| ✅ 正确理由 | ❌ 错误理由 |
|-----------|-----------|
| "海外 QDII 低配 5%，优先加" | "纳指最近很猛，多加" |
| "防御层偏离 -8%，接收止盈资金" | "纯债太无聊了，投科技" |

### 极端场景应对

当出现以下任一系统性危机信号时，按杠铃策略应对：

| 信号 | 参考阈值 |
|------|--------|
| 恐慌指数 | 沪深 300 单周跌 >10%，或单月跌 >20% |
| 流动性危机 | 多只基金暂停赎回 / 大规模限制申购 |
| 债市异常 | 纯债基金单日跌 >0.5%（利率冲击）或连跌 5 天 |
| 地缘冲击 | 战争 / 制裁 / 资本管制升级（以主流媒体报道为准） |

日常波动（单日 ±2%、行业轮动）不触发极端场景。
- 杠铃安全端（纯债/货币）：保持或增持，提供弹药
- 杠铃风险端（科技/商品）：仓位上限锁死在 IPS 目标范围内
- 不做恐慌性清仓，不暂停定投

## 操作护栏

R8（冷却期）、R9（禁止追涨杀跌）和 R10（反脆弱约束）在架构中的位置：
- R8、R9 属于操作护栏 — 每次操作前强制检查
- R10 属于核心层 — 组合构建时一次性满足，后续仅定期复核

### 操作前必问 3 题

1. **「如果我现在是现金，会买入这个组合吗？」**（框架效应纠正）
2. **「理由是什么？里面有"回本""怕跌""别人都买"吗？」**（情绪 vs. 逻辑）
3. **「7 个交易日后，我还会做这个决定吗？」**（冷却期测试）

### 硬门控

任何结构性建议（再平衡、加减定投、止盈）之前，8 项 Checklist 必须已输出：

| # | 检查项 | 诊断问题 | 规则 |
|---|--------|---------|------|
| 1 | 资产分配 | 各类别在目标区间内？ | R4 |
| 2 | 集中度 | 单仓 >20% 或单类 >40%？ | R4 |
| 3 | 定投流向 | 某基金 DCA% > 2× 仓位%？ | R5 |
| 4 | 止盈触发 | 配置偏离 >5%？ | R7 |
| 5 | 亏损审视 | 单仓亏 >10%：逻辑还在吗？（查 references/fund-types.md 投资逻辑速查表） | 补仓规则 |
| 6 | 再平衡 | 上次再平衡 >6 月？各类偏离 >5%？ | R6 |
| 7 | 行为偏差 | 近期操作是否由情绪驱动？ | R8+R9 |
| 8 | 风格漂移 | 组合是否正在赌单一赛道？ | R4+R5 |

## 分析工作流

### 第一步：拉取数据

调用 `scripts/fetch_all_nav.py`（或逐个 curl 天天基金 API）获取最新净值。

**数据时效检查**：读取 `portfolio-state.md` 中的 NAV 日期，如距当前 >7 个自然日，标注 ⚠️「持仓数据可能已过期」，建议用户提供最新截图或确认。

### 第二步：分类诊断

按 IPS 中定义的资产类别对每只基金分类，扫描主要问题：

| 问题 | 检测条件 | 严重度 |
|------|---------|--------|
| 集中度 | 单仓 >20% 或单类超出 IPS 上限 | 🔴 |
| 定投漂移 | DCA% > 2× 仓位% → 组合正被重塑 | 🔴 |
| 重叠 | 多基金跟踪同一指数 | 🟡 |
| 亏损陷阱 | 亏损 >10% 且逻辑已破 | 🟡 |

**定投流向比单仓盈亏更重要**：亏损但 DCA 在稀释 → 自然修复。盈利但 DCA 在加速膨胀 → 制造系统性失衡。

**DCA% 计算公式**：
- DCA% = 该基金月定投金额 ÷ 总月定投金额 × 100
- 仓位% = 该基金市值 ÷ 总组合市值 × 100
- 警报线：DCA% > 2× 仓位% → 该基金正在被定投加速膨胀

### 第三步：输出报告

按深度输出（见下方模板）。

### 三级深度

| 深度 | 触发 | 内容 |
|------|------|------|
| 摘要级 | "怎么样" | Checklist + 需操作项 |
| 标准级 ⭐ | "分析" / 日报 | Checklist + 偏离度 + DCA 流向 + 操作建议 + 行为提示 |
| 深度级 | "深度分析" | 标准 + 逐基诊断(含同类对比) + 完整再平衡方案 + IPS 审查 |

默认标准级。不主动跑深度级。

```
| 代码 | 名称 | 类别 | 昨收 | 估算 | 日涨跌 | 估算市值 | 盈亏 |
|------|------|------|------|------|--------|---------|------|
| ...  |      |      |      |      |        |         |      |
| **合计** | | | | | | **XX,XXX** | **±XX** |
```
数据来源：`fundgz.1234567.com.cn`。每个数字可追溯。

**② 偏离度**
```
📐 各类别 vs IPS 目标
| 类别 | 当前 | 目标 | 偏离 | 状态 |
|------|------|------|------|------|
| 宽基 | XX.X% | XX-XX% | — | ✅ |
| 海外 | XX.X% | XX-XX% | -X.X% | ⚠️ QDII限购 |
```
偏离 ≥3% 标注。结构性原因（QDII 限购）单独说明。

**③ 定投流向**
```
| 基金类别 | 仓位% | DCA% | 趋势 |
|----------|-------|------|------|
| XX类别 | XX.X% | XX.X% | ⚠️ 加速膨胀 |
```
DCA% > 2× 仓位% → ⚠️ 标注。

**④ 行为提示**（按需触发）
```
🧠 连涨 5 日，追涨风险偏高。等回调而非加仓。
```
仅在有触发条件时输出，无触发时跳过。

**⑤ 操作建议**
多数日子：「按兵不动，等定投日」。触发操作条件时给出具体建议（引用规则编号）。

**⑥ 一句话总结**：≤50 字。

### 深度级（额外内容）

在标准级基础上追加：
- 逐只基金诊断（每只 2-3 句，含同类对比）
- 完整再平衡方案（调整前后对比表）
- IPS 符合性逐条审查

## 常见陷阱

1. 不推荐个股（除非明确要求）
2. 区分 C/A 类费率
3. 纳指 QDII 限购是正常的，多只绕开合法
4. 黄金是保险不是增长引擎
5. 商品提供分散化价值而非权益风险溢价
6. 减仓优于清仓
7. macOS 无 `grep -P`，用 Python `re` 模块
8. JSONP 解析用切片 `raw[len("jsonpgz("):-2]`，不用 `lstrip`
9. DCA 方向比仓位快照更重要
10. 数字可追溯：来源不是 API 就是截图，绝无心算
11. 用语忌空泛：不说"基本面良好""前景广阔""值得关注"
12. 中文为主，专用术语（DCA、IPS、QDII、ETF、NAV）保留英文
