# 中国公募基金组合管理

## 触发条件

用户提到以下任一关键词时自动加载：
- **操作类**：分析持仓、再平衡、定投、补仓、调整仓位、止盈、止损
- **监控类**：基金日报、基金周报、净值、涨跌
- **规划类**：投资目标、资产配置、风险承受
- **数据类**：分享基金截图、买入卖出确认

## 首次使用 · IPS 初始化

当 `references/portfolio-state.md` 不存在或含占位符 `XXXXXX` 时，按以下流程主动获取。

### 第一步：询问用户

向用户提问（逐条，每条确认后继续）：

> **① 投资期限**：计划持有多久？（A. <3 年 / B. 3-5 年 / C. 5-10 年 / D. >10 年）
>
> **② 收益目标**：期望的年化回报率？（建议参考：纯债 3-4%、平衡 6-8%、偏股 8-10%）
>
> **③ 最大回撤**：你能承受的最大亏损幅度？（下跌多少%会让你想清仓？建议参考：保守 10-15%、适中 20-25%、进取 30-35%）
>
> **④ 月定投能力**：每月大约能投入多少？
>
> **⑤ 当前持仓**：有哪些基金？请提供代码 + 成本价 + 份额 + 定投设置（或直接发截图）
>
> **⑥ 流动性需求**：未来 3 年内有没有大额支出的计划？（买房、结婚、留学等）
>
> **⑦ 卫星止盈目标**（可选，不设则用默认值）：行业主题基金盈利多少%时分批赎回？
> 建议参考：科技/新能源 25%，消费/医药 20%。（A. 15% / B. 20% / C. 25% / D. 30% / E. 不设止盈）



**基金分类辅助**：拿到用户持仓后，按以下规则自动分类，分类不确定时向用户确认。

| 方法 | 规则 | 示例 |
|------|------|------|
| 名称关键词 | "沪深300/中证500/创业板/上证50/A500/宽基" → 宽基；"纳斯达克/标普/恒生/QDII/全球" → 海外 QDII；"纯债/债券/利率债/信用债/中短债" → 纯债；"红利/股息" → 红利策略；"半导体/芯片/AI/科技" → 行业主题-科技；"医药/医疗/生物/创新药" → 行业主题-医药；"消费/白酒/食品" → 行业主题-消费；"新能源/光伏/电车/碳中和" → 行业主题-新能源；"黄金/金" → 黄金；"资源/有色/原油/能源" → 商品；"货币/现金/余额" → 货币基金；"可转债/转债" → 可转债 |
| 代码前缀 | 16XXXX → QDII；50XXXX/51XXXX/159XXX → ETF；58XXXX → 科创 50 |
| 用户确认 | 名称模糊（如"战略新兴""优选混合"）→ 主动列出分类选项让用户选择 |

### 第二步：矛盾检测

收到用户回答后，**必须**在写入 IPS 前检查以下矛盾：

| 检测项 | 条件 | 矛盾时输出 |
|--------|------|-----------|
| 收益 vs. 回撤 | 目标年化 >10% 且最大回撤 <20% | ⚠️ 历史上年化 10%+ 的组合最大回撤约 30-40%。建议调高回撤容忍到 ≥25%，或降低收益目标到 6-8%。 |
| 收益 vs. 资产结构 | 目标年化 >8% 但防御资产（债+红利）>40% | ⚠️ 防御资产超 40% 的组合年化通常在 5-7%。要么降防御比例，要么降预期。 |
| 时间 vs. 股票仓位 | 投资 <5 年但股票类 >70% | ⚠️ 5 年内需要的钱不应大量投入股票。建议股票类降至 50% 以下。 |
| 定投 vs. 目标 | 月投金额无法在合理时间内达成目标 | ⚠️ 按当前定投速度，达到目标需要 X 年，超出投资期限。建议增加定投金额或调整目标。 |
| 流动性 | 有短期大额支出但组合以股票为主 | ⚠️ 短期需要的资金不应暴露于股市波动。建议单独建立货币基金缓冲池。 |

**矛盾优先处理**：发现矛盾时，让用户选择改哪个参数。不确定时，选更保守的参数。

## 阈值分类

本 skill 中所有阈值分为两类，必须在输出建议时明确区分：

### 专业默认（有学术/行业依据，不建议用户随意修改）

| 阈值 | 值 | 依据 |
|------|-----|------|
| 再平衡 5/25 规则 | 大类别（≥20%）绝对偏离 >5pp；小类别（<20%）相对偏离 >25% | Swedroe 1998, Bogleheads 社区共识 |
| 单仓集中度上限 | ≤20% | 分散化基本原则 |
| 单类集中度上限 | ≤40% | 耶鲁模型多资产分散 |
| DCA 漂移警报 | DCA% > 2× 仓位% | 经验规则（定投正在重塑组合） |
| 卫星层仓位上限 | ≤ 总组合 20% | 塔勒布杠铃策略 |
| 冷却期 | 7 个交易日 | 行为金融（冲动衰减窗口） |
| 极端场景-恐慌 | 沪深 300 单周跌 >10% 或单月跌 >20% | 历史回撤分布 |
| 极端场景-债市 | 纯债单日跌 >0.5% 或连跌 5 天 | 中国债市历史波动 |
| 补仓信号 | 单仓亏损 >10%（仅为逻辑检查信号，非自动操作） | 行为金融（避免追跌陷阱） |

### 用户设定（IPS 初始化时确认，可后续修改）

| 阈值 | 来源 | 合理性检查 |
|------|------|-----------|
| 投资期限 | IPS ① | 越短 → 股票上限越低 |
| 收益目标（年化） | IPS ② | 与回撤容忍交叉验证（>10% 收益需 ≥25% 回撤容忍） |
| 最大回撤容忍 | IPS ③ | 对应参考配置表（investment-philosophy.md） |
| 月定投金额 | IPS ④ | 与目标达成时间交叉验证 |
| 资产配置目标区间 | IPS R4 | 由 R2+R3 推导，检查边界（单类 ≤40%、行业 ≤20%、债券 ≥10%） |
| 再平衡日历日 | IPS R6 | 建议每半年，用户可自定 |
| 卫星止盈目标 | IPS（卫星层） | 行业主题基金默认 20-30%，合理性检查：<15% 过度交易，>50% 可能永不触发 |

**合理性检查原则**：用户设定的阈值若与专业默认冲突或历史数据不支持，agent **必须**在写入 IPS 前提醒，让用户选择改哪个参数。不确定时，选更保守的。

## 架构：核心-卫星双层模型

```
核心层 80-90%（永不择时·永续生效）
├─ 定投纪律 — 自动化扣款，不暂停
├─ 资产配置 — 类别目标区间，不漂移
└─ 再平衡日历 — 每半年/阈值触发

卫星层 10-20%（有条件触发·受核心层约束）
├─ 卫星止盈 — 目标收益触发（仅行业主题，分批赎回）
├─ 再平衡卖出 — 5/25 阈值触发
├─ 补仓 — 仅在逻辑未破 + 配置未超限时
└─ 新资金倾斜 — 优先低配资产（非"看好"的赛道）

冲突解决: 核心层 > 卫星层。R6 优先于卫星止盈（详见卫星层冲突解决表）。
```

---

## 核心层 — 始终生效

### 定投纪律

- 定投 = 用自动化对抗情绪。下跌 = 更多份额。**永不暂停。**
- 支付宝智能定投的浮动扣款范围揭示 base：**base = max ÷ 2**。其他平台的定投参数按用户提供值直接使用，不做换算。
- 限购基金的频次转换：周金额 ÷ 5 = 日金额（或按用户指定的交易日数），不擅自加码。

### 资产配置目标

配置目标由用户 IPS（R4）定义。任何建议引用 R4，不重复定义数字。

**防御层定义**：防御层 = 纯债 + 货币基金 + 红利策略。这三类资产的共性是低波动 + 正息收/股息。黄金和商品不属于防御层——它们波动大、无现金流，属于另类资产。

### 再平衡（R6）

**何时触发**：

- **日历触发**：每半年一次（用户自定日期）
- **阈值触发（5/25 规则）**：
  - 类别占比 ≥20%：绝对偏离 >5 个百分点触发（如目标 30%，超出 25-35% 范围触发）
  - 类别占比 <20%：相对偏离 >25% 触发（如目标 10%，超出 7.5-12.5% 范围触发）

来源：Larry Swedroe《The Only Guide to a Winning Investment Strategy》(1998)，Bogleheads 社区广泛采用。

**如何执行**：

1. 优先用未来 1-2 月新增资金投向低配资产（零成本）
2. 新资金不足以纠正时，卖出超配 → 买入低配
3. 不求完美：偏离修正到阈值以内即可，不必回到精确目标
4. 持有 ≥30 天的 C 类基金赎回费 ¥0（摩擦为零）

**卖出执行（原止盈）**：

当触发阈值且新资金无法在 3 个月内纠正时，执行卖出：
- **不是卖出的理由**："涨太多了""落袋为安""预感要跌"——纯粹的配置纠正
- **卖出后资金去向**：优先转入低配的防御层（纯债/红利），而非闲置或投向热门赛道
- **输出格式**：
```
🔄 再平衡（卖出）
| 基金 | 卖出份额 | 金额(估) | 理由 |
|------|---------|---------|------|
| XXXXXX | ~XXX份 | ~¥X,XXX | XX类别仓位 XX%→~XX%，5/25 阈值触发 R6 |

→ 资金转入：YYYYYY 纯债（防御层低配）
→ 持有天数确认：≥30 天 ✓ | 费用合计：¥0（C类）
```

⚠️ 注意：持续单边上涨时再平衡会跑输不操作——这是降低波动付出的保费，不是 bug。

**优先级矩阵**（偏离以 5/25 规则衡量，非统一 5%）：

| 优先级 | 条件 | 操作 | 窗口 |
|--------|------|------|------|
| 🔴 立即 | 单类偏离超过 5/25 阈值的 2× | 减持超配 → 转防御 | 本周 |
| 🟠 高优 | 单类偏离触及 5/25 阈值 | 调定投流向 | 本月 |
| 🟡 中优 | 定投流向失衡（DCA% > 2× 仓位%） | 降定投基数 | 下个周期 |
| 🟢 低优 | 偏离未触及 5/25 阈值 | 无需操作 | 下次定期 |

## 卫星层 — 有条件触发

卫星层所有操作**必须**满足三个约束，否则退回核心层：
1. 触发条件明确（写在 IPS 里，不是"感觉"）
2. 仓位上限锁死（卫星操作涉及仓位 ≤ 总组合 20%）
3. 有完成标准（不是无限期等待）

### 补仓

**触发条件**：单仓亏损 >10%，且满足以下全部条件：
- ✅ 配置未超限（补仓后该类仍在目标区间内）
- ✅ 逻辑未破（该资产类别的投资逻辑仍成立）
- ❌ 单日跌幅 —— 不是触发条件

**流程**：先检查逻辑是否还在，再看仓位是否允许，最后才建议是否加仓。两步都应输出"逻辑检查"和"仓位检查"结论。

### 卫星止盈（目标收益触发）

**适用范围**：仅限卫星层——行业主题基金（科技、医药、消费、新能源、军工等）。**不适用于**：宽基、债券、红利、黄金、商品。

**为什么需要**：A 股行业主题基金波动极大（最大回撤 40-55%），反复经历「涨 30-50% → 跌回原点」的过山车。在行为金融上极难承受，容易导致追涨杀跌。目标收益止盈从波动中分段锁定利润，降低心理负担。

来源：Wind/天天基金回测数据，中国公募基金行业共识。

**触发条件**（全部满足才触发）：

1. ✅ 单只卫星基金**绝对收益**达到目标阈值
2. ✅ 该类别当前**未低配**（不低于 R4 目标区间下限——避免卖了更偏）
3. ✅ R6 再平衡**未同时触发**（R6 优先于卫星止盈）
4. ✅ 持有 ≥30 天（C 类零赎回费）
5. ❌ 单日涨幅、连涨天数、「感觉太高了」——都不是触发条件

**目标阈值**（用户可调，IPS 初始化时确认）：

| 卫星类型 | 建议目标 | 最低 | 最高 |
|---------|---------|------|------|
| 行业主题-科技（半导体/AI/5G） | 25% | 15% | 40% |
| 行业主题-新能源/光伏 | 25% | 15% | 40% |
| 行业主题-医药/医疗 | 20% | 15% | 35% |
| 行业主题-消费/白酒 | 20% | 15% | 30% |
| 行业主题-军工 | 25% | 15% | 40% |

**合理性检查**：agent 在 IPS 初始化时验证——设太低（<15%）频繁触发、交易成本侵蚀收益；设太高（>50%）可能永远不触发，失去止盈意义。

**执行方式**：

- **分批赎回**：触发后赎 1/3（保守）到 1/2（进取），**不清仓**
- **资金去向**：优先转入低配的防御层（纯债/红利），次选货币基金暂存
- **赎后 DCA 重定向**：卖出后该基金的定投暂停或减半 ≥2 个月，释出资金加至防御层 DCA

**输出格式**：
```
🎯 卫星止盈
| 基金 | 卖出份额 | 金额(估) | 理由 |
|------|---------|---------|------|
| XXXXXX | ~XXX份 | ~¥X,XXX | 盈利 +XX% 触发卫星止盈 (阈值 XX%)，赎 1/3 |

→ 资金转入：YYYYYY 纯债（防御层低配）
→ DCA 调整：XXXXXX 定投暂停 2 月，释出 ~XXX/周加至纯债
→ 持有天数确认：≥30 天 ✓ | 费用合计：¥0（C类）
```

**与 R6 再平衡的冲突解决**：

| 场景 | 处理 |
|------|------|
| R6 与卫星止盈同时触发 | R6 优先。执行 R6 减持方案（量按 R6 要求），顺带完成止盈 |
| 卫星止盈触发但该类别已低配 | 不执行。配置健康 > 利润锁定 |
| 卫星止盈触发但防御层已满 | 资金转入次低配类别（如货币基金），不强行灌入已满的防御层 |
| 多只卫星同时触发止盈 | 可同时执行。但总减持量需检查防御层接收能力 |

### 新资金倾斜

每月定投资金优先投向**当前低配**的资产类别，而非"看好"的赛道。

| ✅ 正确理由 | ❌ 错误理由 |
|-----------|-----------|
| "海外 QDII 低配 5%，优先加" | "纳指最近很猛，多加" |
| "防御层偏离 -8%，接收再平衡卖出或卫星止盈资金" | "纯债太无聊了，投科技" |

### 极端场景应对

当出现以下任一系统性危机信号时（阈值依据见上方「阈值分类 → 专业默认」），按杠铃策略应对：

| 信号 | 参考阈值 |
|------|--------|
| 恐慌指数 | 沪深 300 单周跌 >10%，或单月跌 >20% |
| 流动性危机 | 多只基金暂停赎回 / 大规模限制申购 |
| 债市异常 | 纯债基金单日跌 >0.5% 或连跌 5 天 |
| 地缘冲击 | 战争 / 制裁 / 资本管制升级（以主流媒体报道为准） |

日常波动（单日 ±2%、行业轮动）不触发。
- 杠铃安全端（纯债/货币）：保持或增持，提供弹药
- 杠铃风险端（科技/商品）：仓位上限锁死在 IPS 目标范围内
- 不做恐慌性清仓，不暂停定投

## 操作护栏

R7、R8 每次操作前检查；R9 组合构建时一次性满足，后续定期复核。

### 操作前必问 3 题

1. **「如果我现在是现金，会买入这个组合吗？」**（框架效应纠正）
2. **「理由是什么？里面有"回本""怕跌""别人都买"吗？」**（情绪 vs. 逻辑）
3. **「7 个交易日后，我还会做这个决定吗？」**（冷却期测试）

### 硬门控

任何结构性建议（再平衡、加减定投）之前，8 项 Checklist 必须已输出：

| # | 检查项 | 诊断问题 | 规则 |
|---|--------|---------|------|
| 1 | 资产分配 | 各类别在目标区间内？ | R4 |
| 2 | 集中度 | 单仓 >20% 或单类 >40%？ | R4 |
| 3 | 定投流向 | 某基金 DCA% > 2× 仓位%？ | R5 |
| 4 | 再平衡 | 上次再平衡 >6 月？各类别 5/25 阈值触发？ | R6 |
| 5 | 亏损审视 | 单仓亏 >10%：逻辑还在吗？（查 references/fund-types.md 投资逻辑速查表） | 补仓规则 |
| 6 | 超配卖出 | 单类超配触发 5/25 且新资金无法纠正？ | R6 |
| 7 | 行为偏差 | 近期操作是否由情绪驱动？ | R7+R8 |
| 8 | 风格漂移 | 组合是否正在赌单一赛道？ | R4+R5 |

## 分析工作流

### 第一步：拉取数据

调用 `scripts/fetch_all_nav.py`（或逐个 curl 天天基金 API）获取最新净值。

**数据时效检查**：读取 `portfolio-state.md` 中的 NAV 日期，如距当前 >7 个自然日，标注 ⚠️「持仓数据可能已过期」，建议用户提供最新截图或确认。

### 第二步：分类诊断

按 IPS 中定义的资产类别对每只基金分类，扫描主要问题：

| 问题 | 检测条件 | 严重度 |
|------|---------|--------|
| 集中度 | 单仓 >20% 或单类超出 IPS 上限 | 🔴 |
| 定投漂移 | DCA% > 2× 仓位% → 组合正被重塑 | 🔴 |
| 重叠 | 多基金跟踪同一指数 | 🟡 |
| 亏损陷阱 | 亏损 >10% 且逻辑已破 | 🟡 |

**定投流向比单仓盈亏更重要**：亏损但 DCA 在稀释 → 自然修复。盈利但 DCA 在加速膨胀 → 制造系统性失衡。

**DCA% 计算公式**：
- DCA% = 该基金月定投金额 ÷ 总月定投金额 × 100
- 仓位% = 该基金市值 ÷ 总组合市值 × 100
- 警报线：DCA% > 2× 仓位% → 该基金正在被定投加速膨胀

### 第三步：输出报告

按深度输出（见下方模板）。

### 三级深度

| 深度 | 触发 | 内容 |
|------|------|------|
| 摘要级 | "怎么样" | Checklist + 需操作项 |
| 标准级 ⭐ | "分析" / 日报 | Checklist + 偏离度 + DCA 流向 + 操作建议 + 行为提示 |
| 深度级 | "深度分析" | 标准 + 逐基诊断(含同类对比) + 完整再平衡方案 + IPS 审查 |

默认标准级。不主动跑深度级。

## 输出模板

### 表格格式规范（强制）

- **全篇统一使用 Markdown `|` 管道表**。禁止使用 `┃ ┏ ┓ ┗ ┛ ━` 等 box-drawing 字符画表。
- **合计行紧接数据行**：`| **合计** |` 行与上方最后一行数据之间**不加空行**，否则表格断裂为两张表。
- **各章节（①-⑦）之间加一个空行**，避免内容粘连。
- **中英文混排对齐**：左对齐即可，不需要手动填充空格对齐源码。
- **不画 ASCII 框线**：不要用 `+--+--+` 或 `┏━┳━┓` 之类的字符装饰表格。

### 标准级（日报/周报）

**① 持仓快照**
```
| 代码 | 名称 | 类别 | 成本 | 最新NAV | 日涨跌 | 市值(估) | 盈亏 | 盈亏% |
|------|------|------|------|---------|--------|----------|------|-------|
| XXXXXX | 某某沪深300联接 | 宽基 | X.XXXX | X.XXXX | +X.XX% | XX,XXX | ±XXX | ±X.XX% |
| **合计** | | | | | | **XX,XXX** | **±X,XXX** | **±X.XX%** |
```
数据来源：`fundgz.1234567.com.cn`。每个数字可追溯。表头用中文简写（代码、名称、类别、成本、NAV、涨跌、市值、盈亏）。

**② 偏离度**
```
📐 各类别 vs IPS 目标
| 类别 | 当前% | 目标 | 偏离 | 状态 |
|------|-------|------|------|------|
| 宽基 | 35.4% | 30-45% | — | ✅ 居中 |
| 海外 | 12.6% | 15-25% | -2.4% | ⚠️ 低于下限（QDII限购） |
```
偏离 ≥3% 标注。结构性原因（QDII 限购）单独说明。状态列用简短中文+emoji。

**③ 定投流向**
```
| 类别 | 仓位% | DCA% | DCA/仓位比 | 趋势 |
|------|-------|-------|-----------|------|
| A股科技 | 17.7% | 44.5% | 2.51× 🔴 | 严重加速膨胀 |
| 宽基 | 35.4% | 29.6% | 0.84× | 温和稀释 |
```
DCA% > 2× 仓位% → 🔴 标注。DCA/仓位比列可选，仅在触发警报时展示。

**④ 行为提示**（按需触发）
```
🧠 连涨 5 日，追涨风险偏高。等回调而非加仓。
```
仅在有触发条件时输出，无触发时跳过。

**⑤ Checklist**（任何结构性建议前必输出）
```
| # | 检查项 | 诊断 | 状态 |
|---|--------|------|------|
| 1 | 资产分配 | QDII 低配 2.4%（限购）、另类略超上限 | 🟡 |
| 2 | 集中度 | 最大单仓 18.5%、单类 35.4%，安全 | ✅ |
```
每行诊断 ≤30 字。状态统一用：✅ 正常 / 🟡 注意 / 🔴 需操作 / 🟢 无操作 / ⚠️ 趋势警告。

**⑥ 操作建议**
```
| 优先级 | 操作 | 规则 | 窗口 |
|--------|------|------|------|
| 🔴 | XXXXXX 减持 1/3 → 转入纯债 YYYYYY | R6 | 本周 |
| 🟠 | ZZZZZZ DCA 从 XXX→XXX/周 | R5 | 下次定投前 |
| 🟡 | WWWWWW 盈利 +XX%，赎 1/3 → 转纯债 | 卫星止盈 | 本月 |
| 🟢 | 黄金/资源不动，红利 DCA 维持 | 逻辑未破 | — |
```
多数日子：「按兵不动，等定投日」。触发操作条件时给出具体建议（引用规则编号）。R6 和卫星止盈同时触发时仅输出 R6 条目。

**⑦ 一句话总结**：≤50 字。

### 深度级（额外内容）

在标准级基础上追加：
- **逐只基金诊断**：每只 2-3 句，不含嵌套表格。格式为「代码 名称 · 类别 · 仓位%」+ 诊断文字。
- **完整再平衡方案**：调整前后对比表（`|` 管道表）。
- **IPS 符合性逐条审查**：逐条对照 R1-R9，格式同 Checklist 表。

## 常见陷阱

### 投资原则

1. 不推荐个股（除非明确要求）
2. 纳指 QDII 限购是正常的，多只绕开合法
3. 黄金是保险不是增长引擎
4. 商品提供分散化价值而非权益风险溢价
5. 减仓优于清仓
6. DCA 方向比仓位快照更重要（见分析工作流-第二步）

### 操作细节

7. 区分 C/A 类费率：定投/短持选 C（≥30 天零赎回费），大额/长持选 A
8. 数字可追溯：来源不是 API 就是截图，绝无心算

### 输出规范

9. 用语忌空泛：不说"基本面良好""前景广阔""值得关注"
10. 中文为主，专用术语（DCA、IPS、QDII、ETF、NAV、REITs）保留英文
